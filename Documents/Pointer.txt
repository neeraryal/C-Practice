    <div class="container">
        <h1>POINTER CONST-CORRECTNESS</h1>
        <p class="subtitle">Master Guide for C Programming</p>

        <div class="section">
            <h2>1. THE RIGHT-TO-LEFT RULE</h2>
            <p>Read pointer declarations from right to left.</p>
            
            <div class="code-block"><code>int *p
<span class="comment">    // p is a pointer to int</span>

const int *p
<span class="comment">    // p is a pointer to const int
    // (data immutable via p)</span>

int * const p
<span class="comment">    // p is a const pointer to int
    // (address immutable)</span>

const int * const p
<span class="comment">    // p is a const pointer to const int
    // (both locked)</span></code></div>
        </div>

        <div class="section">
            <h2>2. THE LOGIC OF "PERMISSIONS"</h2>
            <div class="info">
                <strong>Key Concept:</strong> const is a restriction on the ACCESS PATH, not necessarily on memory.
            </div>
            
            <div class="code-block"><code>int x = 10;
const int *p = &x;

*p = 20;   <span class="comment">// ERROR</span>
x  = 20;   <span class="comment">// OK</span></code></div>

            <p>x is mutable. p only provides a <strong>read-only VIEW</strong> of x.</p>

            <h3>Two kinds of const:</h3>
            <ul>
                <li><strong>Physical const:</strong> <code>const int x;</code> — Object itself is immutable (may live in ROM).</li>
                <li><strong>Logical const:</strong> <code>const int *p;</code> — Compiler prevents mutation THROUGH p only.</li>
            </ul>
        </div>

        <div class="section">
            <h2>3. DOUBLE POINTERS (**)</h2>
            <p>With **, there are <strong>THREE independent levels</strong> of mutability.</p>
            
            <div class="rule">
                <ul>
                    <li><strong>Level 1:</strong> **pp → final data</li>
                    <li><strong>Level 2:</strong> *pp → inner pointer</li>
                    <li><strong>Level 3:</strong> pp → outer pointer</li>
                </ul>
            </div>

            <h3>Examples:</h3>
            <div class="code-block"><code>const int **pp
<span class="comment">    // Data protected
    // Inner pointer mutable
    // DANGEROUS (breaks const safety)</span>

int * const *pp
<span class="comment">    // Inner pointer protected
    // Data mutable</span>

int ** const pp
<span class="comment">    // Outer pointer protected</span></code></div>
        </div>

        <div class="section">
            <h2>4. CRITICAL WARNINGS</h2>
            
            <div class="warning">
                <strong>A) Dropping const (ILLEGAL)</strong>
                <div class="code-block"><code>const int *cp;
int *p = cp;  <span class="comment">// ERROR (removes protection)</span></code></div>
            </div>

            <div class="warning">
                <strong>B) Casting away const</strong>
                <div class="code-block"><code>(int *)p</code></div>
                <p>This bypasses safety checks. Using it to modify data originally declared const is <strong>UNDEFINED BEHAVIOR</strong>.</p>
            </div>

            <div class="warning">
                <strong>C) Double-pointer security rule</strong>
                <div class="code-block"><code>int **  -> const int **   <span class="comment">// NOT allowed</span></code></div>
                <p><strong>Reason:</strong> Prevents a backdoor where const data could be mutated through pointer re-routing.</p>
            </div>
        </div>

        <div class="section">
            <h2>5. FUNCTION PARAMETERS</h2>
            
            <h3>Read-only guarantee:</h3>
            <div class="code-block"><code>void f(const int *p);</code></div>
            <p><strong>Meaning:</strong> Function promises not to modify caller data.</p>

            <h3>Const pointer parameter:</h3>
            <div class="code-block"><code>void f(int * const p);</code></div>
            <p>Usually <strong>useless noise</strong>. Pointers are already passed by value.</p>
        </div>

        <h1 style="margin-top: 80px;">C POINTERS — CORE FOUNDATION</h1>
        <p class="subtitle">Essential Concepts</p>

        <div class="section">
            <h2>1. WHAT A POINTER IS</h2>
            <p>A pointer is a variable that stores an <strong>ADDRESS</strong>.</p>
            <div class="info">
                Pointer type describes WHAT lives at the address, not the address itself.
            </div>
            <div class="code-block"><code>int x;
int *p = &x;</code></div>
        </div>

        <div class="section">
            <h2>2. DEREFERENCING (*)</h2>
            <div class="code-block"><code>p   <span class="comment">// address</span>
*p  <span class="comment">// value at address</span>

<span class="comment">// * on RHS  -> read
// * on LHS  -> write</span>

*p = 10;   <span class="comment">// write</span>
x  = *p;   <span class="comment">// read</span></code></div>
        </div>

        <div class="section">
            <h2>3. PASS-BY-VALUE (NO EXCEPTIONS)</h2>
            <div class="rule">
                C ALWAYS passes by value.
            </div>
            <ul>
                <li><code>int</code> → value copied</li>
                <li><code>int *</code> → address copied</li>
            </ul>
            <p>Data changes work because multiple pointers refer to the same memory.</p>
            <p>Pointer reassignment does NOT affect caller.</p>

            <h3>To modify caller's pointer:</h3>
            <div class="code-block"><code>void f(int **p) {
    *p = NULL;
}</code></div>
        </div>

        <div class="section">
            <h2>4. POINTER ARITHMETIC</h2>
            <div class="info">
                Pointer increments scale by sizeof(type).
            </div>
            <div class="code-block"><code>p + 1 != address + 1 byte

*(p + i) == p[i]</code></div>
        </div>

        <div class="section">
            <h2>5. ARRAYS VS POINTERS</h2>
            <p>Array name decays to pointer to first element.</p>
            <div class="warning">
                <strong>BUT:</strong> <code>sizeof(array) != sizeof(pointer)</code>
            </div>
        </div>

        <div class="section">
            <h2>6. NULL POINTER</h2>
            <p>NULL means "points to nothing valid".</p>
            <div class="warning">
                Dereferencing NULL ⇒ <strong>UNDEFINED BEHAVIOR</strong>
            </div>
        </div>

        <h1 style="margin-top: 80px;">CONST WITH POINTERS</h1>
        <p class="subtitle">Most Important Section</p>

        <div class="section">
            <h2>7. CONST APPLIES TO ACCESS PATH</h2>
            <div class="code-block"><code>int x;
const int *p = &x;</code></div>
            <p>x is mutable. Access through p is read-only.</p>
        </div>

        <div class="section">
            <h2>8. SINGLE-LEVEL CONST MATRIX (MEMORIZE)</h2>
            <div class="matrix">
                <table>
                    <tr>
                        <th>Declaration</th>
                        <th>Pointer</th>
                        <th>Data</th>
                    </tr>
                    <tr>
                        <td><code>int *p</code></td>
                        <td>mutable</td>
                        <td>mutable</td>
                    </tr>
                    <tr>
                        <td><code>const int *p</code></td>
                        <td>mutable</td>
                        <td>const</td>
                    </tr>
                    <tr>
                        <td><code>int * const p</code></td>
                        <td>const</td>
                        <td>mutable</td>
                    </tr>
                    <tr>
                        <td><code>const int * const p</code></td>
                        <td>const</td>
                        <td>const</td>
                    </tr>
                </table>
            </div>
        </div>

        <div class="section">
            <h2>9. CONST CONVERSION RULE</h2>
            <div class="rule">
                <p>✅ You may <strong>ADD</strong> const.</p>
                <p>❌ You may <strong>NEVER REMOVE</strong> const.</p>
            </div>
            <div class="code-block"><code>int *        -> const int *   <span class="comment">// OK</span>
const int *  -> int *         <span class="comment">// ERROR</span></code></div>
        </div>

        <div class="section">
            <h2>10. WHY POINTER-TO-POINTER (**)</h2>
            <p>Required when a function must:</p>
            <ul>
                <li>Allocate memory</li>
                <li>Free memory and NULL the pointer</li>
                <li>Reassign ownership</li>
                <li>Update list / tree heads</li>
            </ul>
            <div class="code-block"><code>void alloc(int **p);</code></div>
        </div>

        <div class="section">
            <h2>11. CONST WITH ** (CRITICAL RULE)</h2>
            <div class="warning">
                <strong>NEVER DO THIS:</strong>
                <div class="code-block"><code>const int **pp;</code></div>
                <p><strong>Why:</strong></p>
                <ul>
                    <li>Allows pointer re-routing</li>
                    <li>Breaks const guarantees</li>
                    <li>Compiler cannot enforce safety</li>
                </ul>
            </div>

            <div class="info">
                <strong>CORRECT, SAFE FORM:</strong>
                <div class="code-block"><code>const int * const *pp;</code></div>
                <p><strong>Meaning:</strong></p>
                <ul>
                    <li>Data protected</li>
                    <li>Inner pointer protected</li>
                    <li>Const guarantee preserved</li>
                </ul>
            </div>
        </div>

        <div class="section">
            <h2>12. GOLDEN RULE FOR MULTI-LEVEL CONST</h2>
            <div class="rule">
                <strong>If the DATA is const, EVERY pointer level above it must also be const.</strong>
            </div>
        </div>

        <div class="section">
            <h2>13. FINAL MENTAL MODEL</h2>
            <ul>
                <li>C always copies values.</li>
                <li>Pointers work because values happen to be addresses.</li>
                <li>const restricts ACCESS PATHS, not memory itself.</li>
                <li>Const safety fails when pointer re-routing is allowed.</li>
            </ul>
        </div>
